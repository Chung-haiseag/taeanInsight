# 평가셋 검증 가이드 — 태안신문 편집부용

## 한 줄 요약

`checklist.md`의 36문항을 검토하여 정답·출처가 맞는지 확인하고, 결과 컬럼에 `O / △ / X`를 표시해주세요. 완료된 파일을 저에게 전달하면 자동으로 평가셋이 갱신됩니다.

---

## 왜 이 작업이 필요한가요?

태안 AI 인텔리전스 커먼즈 플랫폼은 Phase 1(2026-05~06)에 3종 LLM(Together AI Solar Mini, Anthropic Claude Haiku, 참고용 Llama-3.1-8B)을 비교 평가해 최종 선정합니다. 평가의 객관성은 **정답이 정확해야** 보장됩니다.

Claude가 1차로 작성한 50문항 중 14문항은 공개 자료로 즉시 검증되었지만, **나머지 36문항은 태안 지역 전문 지식이 있는 편집부의 확인**이 필요합니다. 정답이 틀린 상태로 벤치마크하면 "맞은 답을 틀렸다"고 채점하는 일이 발생합니다.

        ↓

## 검증 표기법

### `O` — 정답·출처 모두 정확
- 추가 작업 없음
- `verification_status: verified`로 갱신

### `△` — 일부 수정 필요
- 결과 컬럼에 `△` 표시
- **수정사항** 컬럼에 변경 내용 기입 (아래 양식 참조)

### `X` — 사실 오류·삭제 권장
- 출처를 찾을 수 없거나, 명백히 틀린 경우
- 해당 문항은 평가셋에서 제거됨
- Phase 1 종료(6/30) 전 대체 문항으로 보충

---

## 수정사항 양식

수정사항 컬럼에는 다음 형식으로 기입:

### 정답 변경
```
ANSWER: 안면도 자연휴양림은 안면송이 우거진 약 450ha 규모의 송림으로 ...
```

### 키워드 추가/제거
```
KEYWORDS: +몽산포 +기지포 -안면도
```
- `+키워드`: 추가
- `-키워드`: 제거

### 출처 보강
```
SOURCE: 문화재청 고시 제2002-3호 (2002.11.06 지정)
```

### 여러 항목 동시 변경
줄바꿈으로 구분:
```
ANSWER: 새 정답...
KEYWORDS: +새키워드
SOURCE: 새 출처
```

## 검증 시 자주 묻는 질문

### Q. 정답은 맞는데 키워드가 부족해요.
A. `△`로 표시하고 `KEYWORDS: +추가키워드` 형식으로 보강해주세요.

### Q. 정답은 맞는데 출처가 너무 추상적이에요.
A. `△`로 표시하고 `SOURCE: 구체적인 출처` 기입.

### Q. 질문 자체를 수정하고 싶어요.
A. 질문 수정이 필요한 경우 `△` + `수정사항`에 직접 변경 사항 기입. 단, 답이 크게 달라지면 새 문항으로 추가하는 게 깔끔합니다 (이 경우 X 처리 후 별도 알림).

### Q. 같은 도메인 다른 문항에 더 좋은 질문이 떠올랐어요.
A. `X` 처리 + 별도 메모로 새 문항 후보 제안. Claude가 새 문항을 만들어 추가하겠습니다.

### Q. 출처가 유료 자료(저작권)인 경우?
A. 그대로 출처 표기는 가능하지만, 답변에는 인용 출처를 짧게 요약만 (저작권법 정당한 인용 범위).

## 검증 결과 처리 (작업 완료 후)

편집부에서 검증 완료 후 다음 명령으로 JSON 일괄 갱신:

```bash
cd /Applications/taean/eval/verification
python apply_verification.py checklist_completed.md
```

스크립트가 다음을 처리:
- `O` 표시된 항목 → `verification_status: verified`
- `△` 표시된 항목 → 수정사항 적용 + `verification_status: verified`
- `X` 표시된 항목 → JSON에서 제거

처리 결과는 `verification_log.json`에 기록되어 추적 가능.
